Create or update the founder's `raise-brief.md` in the project root.

This brief is the single highest-leverage file in the setup. It is read into every session automatically by the plugin's SessionStart hook, so every command starts from the founder's actual raise instead of a blank page. Explain this to the founder when you write it.

If `raise-brief.md` already exists in the project root:
- Read it first and update it in place rather than starting over.

Interview the founder for each section (ask only for what is missing or stale):
1. **What you are raising** — stage, round size, the valuation range you are anchoring to.
2. **Traction** — revenue, growth rate, the two or three logos or numbers that matter.
3. **Ideal investor** — stage focus, sector, check size wanted from a lead and from a follow.
4. **Rules and tone** — lines you will not cross on terms, sectors you will not take money from, the tone you want in every message.

Then write `raise-brief.md` in the project root as clean, scannable markdown with one section per heading above. Confirm the path you wrote to and remind the founder it will be auto-loaded each session.
