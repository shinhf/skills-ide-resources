# Designing a Slidedoc -- Layout, Templates & Visual Guidelines

> **Source:** Nancy Duarte's *Slidedocs* ebook (Chapter: "Designing a Slidedoc"),
> Duarte templates, and Duarte design blog posts.

## Template Types

Duarte provides three main layout types for Slidedoc pages:

### 1. Standard Layout
- The workhorse page for regular content
- Balanced ratio of text and visuals
- Includes headline, body text area, and space for a supporting visual
- Used for the majority of content pages

### 2. Impact Layout
- Designed for pages that need to make a strong visual statement
- Larger visual area, less text
- Used for key findings, major data visualizations, or section highlights
- The "billboard" of the Slidedoc

### 3. Section Layout
- Divider pages that introduce new sections or chapters
- Minimal content: section title and optional subtitle/description
- Creates visual breathing room between content sections
- Helps readers navigate the document structure

## Layout Design Principles

### The Information-to-Whitespace Ratio
- Maintain the **right balance** of information to whitespace for quick processing
- Too much content = cognitive overload
- Too much whitespace = wasted opportunity and fragmented reading
- Aim for approximately **40-60% content, 40-60% whitespace** (varies by layout type)

### Grid-Based Design
- Use a consistent **grid system** for element placement
- Align text, images, charts, and callouts to the grid
- Consistent alignment creates invisible guides that improve flow
- Templates enforce grid consistency automatically

### Visual Hierarchy Through Layout
- **Position** communicates importance (top-left = most important in LTR languages)
- **Size** communicates importance (larger = more important)
- **Isolation** communicates importance (elements with whitespace around them stand out)
- Use these spatial cues deliberately

## Design Elements

### Color Usage
- **Limited palette**: 3-5 colors maximum for the entire document
- **Brand colors**: Use consistently for headings, accents, and highlights
- **Neutral base**: Gray tones for body text, gridlines, and supporting elements
- **Accent color**: One bright/saturated color for emphasis (call-to-action, key data)
- **Consistent meaning**: Same color = same meaning throughout the document

### Typography
- **1-2 font families** maximum (e.g., one sans-serif for headings, one for body)
- **Clear size hierarchy**: Title > Section Header > Body > Caption
- **Readable body text**: Minimum ~10-11pt for comfortable reading
- **Sans-serif fonts** work well for screen reading (the primary consumption mode)
- **Consistent styling**: Same font/size/color for same-level elements throughout

### Imagery and Icons
- Use images that **reinforce the message**, not just decorate
- **Consistent image treatment**: All photos same style (e.g., all with rounded corners, all with same filter)
- **Icons and illustrations**: Use a unified icon set (same style, weight, color)
- Duarte's templates include a **graphic toolkit** with consistent iconography
- Avoid clip art or mismatched visual styles

### Charts and Data Visualization
- **Show conclusions**, not just data: help the reader understand "so what?"
- Choose chart type based on the story:
  - **Bar charts**: Comparing categories
  - **Line charts**: Showing trends over time
  - **Pie charts**: Showing proportions of a whole (limit to 5-6 segments)
  - **Tables**: Precise values for comparison
- **Color strategy for data**:
  - Light neutral colors (gray) for scales, gridlines, tick marks
  - Bright/accent color for the key data point or trend you want to emphasize
  - Don't make every data series a different bright color
- **Label directly** on charts when possible (reduce legend dependency)
- Include a **clear title** that states the conclusion, not just the metric

## Page Composition Checklist

For each Slidedoc page, verify:

- [ ] **One main idea** clearly communicated
- [ ] **Headline** states a conclusion (not a topic label)
- [ ] **Visual hierarchy** is clear (what's most important is visually dominant)
- [ ] **Whitespace** is adequate (page doesn't feel cramped)
- [ ] **Text and visuals** work together (not competing or redundant)
- [ ] **Color usage** is consistent with the rest of the document
- [ ] **Fonts** match the established hierarchy
- [ ] **Alignment** follows the grid
- [ ] **Flow** guides the eye in a logical reading order
- [ ] Passes the **Glance Test** (main message identifiable in 3 seconds)

## Avoiding Common Design Mistakes

| Mistake | Fix |
|---------|-----|
| Wall of text with no visuals | Break into smaller blocks; add a supporting visual |
| Too many fonts/colors | Limit to 2 fonts, 3-5 colors |
| Decorative but meaningless images | Replace with relevant diagrams or data |
| Every element is the same size/weight | Create clear hierarchy with size/color/position |
| Inconsistent styling across pages | Use templates; check unity across the full document |
| Charts without context | Add headline conclusions and callout annotations |
| Cluttered page with no whitespace | Remove or redistribute content across pages |
| Random element placement | Align to grid; follow consistent layout patterns |

## Duarte Template Details

Duarte provides **3 free PowerPoint templates** (Theme 1, Theme 2, Theme 3), each containing:

- **Standard, Impact, and Section layouts** as described above
- A **graphic toolkit** with consistent iconography, shapes, and visual elements

with content areas sized for approximately **75-175 words per page**.
