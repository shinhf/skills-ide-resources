# Slidedoc Review Checklist -- Quality Evaluation Criteria

> **Source:** Synthesized from Nancy Duarte's five design principles, Glance Test,
> critique language framework, and Slidedoc best practices.

## Purpose

This checklist provides structured criteria for evaluating whether a Slidedoc meets
Duarte's best practices. Use it to review individual pages and the document as a whole.

---

## Part 1: Per-Page Review

### A. Message Clarity

- [ ] The page communicates **one main idea**
- [ ] The **headline states a conclusion** (not a topic label)
- [ ] The main message is understandable **without reading body text**
- [ ] Supporting content **reinforces** (not distracts from) the main message
- [ ] The page passes the **Glance Test** (main point identifiable in 3 seconds)

### B. Contrast

- [ ] The most important element is **visually dominant** (larger, bolder, or brighter)
- [ ] There is clear **differentiation** between heading, body, and caption text
- [ ] Key data points or findings are **highlighted** with accent color or emphasis
- [ ] Prioritization is **clear, not indistinct**

### C. Whitespace

- [ ] The page has **adequate breathing room** (not cramped or cluttered)
- [ ] Whitespace is used to **group related elements** and **separate unrelated ones**
- [ ] Margins are **generous and consistent**
- [ ] The page does not feel like a Word document pasted into PowerPoint

### D. Hierarchy

- [ ] A clear **parent/child relationship** exists between ideas
- [ ] The visual treatment of elements reflects their **importance**
- [ ] The reader can identify the **most important takeaway** immediately
- [ ] Supporting details are visually subordinate to the main message

### E. Unity (checked per page but evaluated document-wide)

- [ ] Color usage is **consistent** with the document's palette
- [ ] Typography follows the **established hierarchy** and font choices
- [ ] Image/icon style is **consistent** with other pages
- [ ] Element placement matches the **template/grid patterns**

### F. Flow

- [ ] There is a **clear, logical reading path** on the page
- [ ] The eye moves naturally from the most important to least important elements
- [ ] Visual cues (alignment, numbering, arrows) **guide the reader**
- [ ] No elements cause the eye to **jump randomly** across the page

### G. Text Quality

- [ ] Text is **concise and skimmable** (not wall-of-text)
- [ ] Bullet points are used for **3+ related items**
- [ ] Paragraphs are **short** (3-5 sentences maximum)
- [ ] Language is **clear and jargon-free** (or jargon is audience-appropriate)
- [ ] Conclusions and next steps are **explicitly stated**

### H. Visual Quality

- [ ] Visuals **reinforce the message** (not decorative filler)
- [ ] Charts use the **right chart type** for the data story
- [ ] Charts have **conclusion-driven titles** (not metric labels)
- [ ] Data visualization uses **neutral backgrounds** with **accent highlights**
- [ ] Labels are **applied directly** to charts (minimal legend dependency)
- [ ] Images are **high quality** and consistently styled

---

## Part 2: Full-Document Review

### I. Structure

- [ ] Document has a **clear table of contents** (if > 10 pages)
- [ ] An **executive summary** captures key findings and recommendations
- [ ] Sections are **logically organized** with clear section dividers
- [ ] The document **tells a coherent story** from beginning to end
- [ ] **Recommendations and next steps** are clearly stated

### J. Unity (Document-Wide)

- [ ] **Color palette** is limited (3-5 colors) and consistent throughout
- [ ] **Typography** uses 1-2 font families consistently
- [ ] **Layout templates** are used consistently (standard, impact, section)
- [ ] **Image treatment** is uniform (same style/filter/sizing approach)
- [ ] **Element placement** is predictable (titles, page numbers, logos)
- [ ] The document looks like it comes from **one author/source**

### K. Overall Flow

- [ ] The document has a **logical progression** of ideas
- [ ] Section transitions are **smooth** (reader understands why topics are ordered this way)
- [ ] The level of detail is **appropriate and consistent** across sections
- [ ] The document can be **skimmed** (reading only headlines tells the core story)
- [ ] The document can be **deep-read** (body text adds genuine value)

### L. Audience Appropriateness

- [ ] The level of detail matches the **audience's needs** (executive vs. operational)
- [ ] Technical complexity is **calibrated** to the audience's expertise
- [ ] The document stands alone (understandable **without a presenter**)
- [ ] Key terms are defined or **audience-appropriate**
- [ ] The "ask" or call-to-action is **clear and actionable**

## Severity Ratings for Issues

When reviewing, classify issues by severity:

| Severity | Description | Example |
|----------|-------------|---------|
| **Critical** | Fundamentally breaks comprehension | No clear message on a page; wall of text with no hierarchy |
| **Major** | Significantly reduces effectiveness | Poor chart choice; competing messages on one page |
| **Minor** | Suboptimal but doesn't block understanding | Slightly inconsistent font size; minor alignment issue |
| **Suggestion** | Improvement opportunity | Could add a callout box; color could be more strategic |

## Quick Evaluation Heuristic

For a rapid assessment, score these five areas 1-5:

1. **Skimmability**: Can I understand the full story by reading only headlines? (1-5)
2. **Visual clarity**: Are visuals clear, relevant, and well-designed? (1-5)
3. **Consistency**: Does the document look unified and professional? (1-5)
4. **Message strength**: Are conclusions and recommendations clear? (1-5)
5. **Standalone quality**: Would this make sense without a presenter? (1-5)

**Total: /25** -- Scores below 15 indicate significant revision needed.
